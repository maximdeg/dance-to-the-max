# Three dance-gated cumulative tiers (revises the spec's "4 packages")

Pricing uses three subscription **Tiers**, not four packages, and access is gated by **Dance** (style), not by difficulty **Level**. Tiers form a cumulative ladder (T1 ⊂ T2 ⊂ T3): each higher Tier unlocks more Dances. Each Dance carries a configurable minimum Tier set by the Super Admin, so the lineup is data-driven rather than hardcoded. Rationale: it matches how the studio sells access ("unlock more dances") and keeps the access check trivial — "is this Video's Dance in the Subscriber's Tier?".
